# JB 에이전트

## 1. 서비스명

**미정**

## 2. 서비스 한 줄 소개

외국인 고객이 “월급 계좌 만들거나 지점예약 준비해줘”라고 말하면, AI Agent가 **서류 확인 → 계좌개설 경로 판단 → KYC/AML 점검 → 지점예약/상담연결**까지 실행하는 서비스

## 3. 아이디어 배경

JB금융은 외국인등록증 발급 전에도 여권·생체인증 기반으로 비대면 계좌개설을 추진 중이다.  
즉, 외국인 금융 진입장벽 해결은 JB금융과 직접 연결된다.

## 4. 기존 방법과 한계

| 기존 방식 | 한계 |
|---|---|
| 홈페이지/FAQ | 고객이 직접 찾아야 함 |
| 단순 챗봇 | 답변은 하지만 업무를 끝내주지는 못함 |
| 콜센터/지점 방문 | 언어·서류 누락·재방문 문제가 큼 |
| 직원 수동 확인 | 상담 품질과 처리 속도가 직원별로 달라짐 |

## 5. 개발 목표

외국인 고객의 초기 금융 정착 과정을 **상담이 아니라 실행 중심**으로 자동화한다.

| 목표 | 내용 |
|---|---|
| 금융 진입 자동화 | 계좌개설 가능 경로를 Agent가 판단 |
| 업무 실행 | 서류 체크, 지점 예약, 상담 연결까지 수행 |
| 안전한 처리 | KYC/AML, 준법 검토, 감사 로그를 함께 남김 |

## 6. LLM과 다른 AI Agent 활용 시나리오

고객이 이렇게 입력한다.

> “한국에 막 왔고, 월급 받을 계좌와 본국 송금 준비를 하고 싶어요.”

단순 LLM은 필요한 서류를 설명한다.

**JB 에이전트**은 다르게 움직인다.

| 단계 | Agent 실행 |
|---|---|
| 1 | 고객 목표를 “계좌개설 + 송금 준비 + 생활결제 세팅”으로 분해 |
| 2 | 여권, 체류자격, 모바일 외국인등록증 가능 여부 확인 |
| 3 | 계좌개설 가능 경로를 비대면/지점방문/추가서류 필요로 분기 |
| 4 | KYC/AML Agent가 위험 여부 점검 |
| 5 | Compliance Agent가 안내 문구와 절차를 검토 |
| 6 | Action Agent가 지점 예약·상담 연결·서류 체크리스트 생성을 수행 |
| 7 | Audit Agent가 판단 근거와 처리 이력을 저장 |

## 7. 타겟 사용자

| 사용자 | 필요 |
|---|---|
| 외국인 근로자 | 월급 계좌, 본국 송금 |
| 외국인 유학생 | 생활비 계좌, 체크카드, 등록금 납부 |
| 장기체류 외국인 | 통신·공과금·생활결제 세팅 |
| 은행 직원 | 외국인 고객 응대 절차 자동 정리 |

## 8. 기대 효과

| 대상 | 효과 |
|---|---|
| 외국인 고객 | 금융생활 시작 시간이 짧아짐 |
| 직원 | 반복 상담과 서류 확인 부담 감소 |
| JB금융 | 외국인 고객 시장을 선점할 수 있음 |
| 리스크 관리 | KYC/AML·준법·감사 로그 기반으로 안전성 확보 |
| 서비스 확장 | 계좌개설 이후 송금, 카드, 대출, 자산관리로 확장 가능 |

## 9. 핵심 문장

> JB 에이전트는 외국인 고객에게 금융 정보를 알려주는 챗봇이 아니라, 외국인의 초기 금융 정착 업무를 여러 Agent가 나눠 실행하는 JB금융형 Agentic Banking 서비스다.
